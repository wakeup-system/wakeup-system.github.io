Targets

The WakeUp System's Targets.

Once a trigger sends a singal, the WakeUp system will map the correct target device to that trigger. The target devices utilised include the Home Assistant API to connect Matter devices, WakeOnLAN to wake up displays, and the Telegram API sends alerts on a registered mobile device.

Home Assistant API

This API allows us to control Matter devices within the local network.
To interact with Matter devices, the Home Assistant integration utilises its "Matter controller". Operating separately, Matter server add-ons enable the functioning of the controller software, creating a link between the Matter network and Home Assistant. Communication between Home Assistant Matter integration and the server occurs through a WebSocket.
The Matter devices our team uses include smart devices such as lights and smart plugs. Based on the trigger that the patient has requested, the WakeUp System will map the relevant target device to it. Additionally, media players are another target device.
WakeOnLAN

This has been developed to allow patients to power on TVs or other computer stations on the network using a Python package. The target device must be be in a low-power state (sleeping). The target device will be waiting for a specific “Magic Packet” to be sent. This packet is then transmitted to the broadcast address of the local network. Upon receiving the packet, the target device will validate it and wake up if the packet contains all the accurate information, including the MAC address and IPv4 address.
Telegram API Bot

This is a custom bot used to communicate on alerts. Once activated, the bot will send alerts every 5 seconds to the user, and it will continue to run until the user stops it from the message channel.
